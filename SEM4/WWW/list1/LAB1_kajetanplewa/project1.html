    public String dehash(String password,byte[] salt, int iterations) throws NoSuchAlgorithmException, InvalidKeySpecException {
        KeySpec spec = new PBEKeySpec(password.toCharArray(),salt,iterations, 128);
        SecretKeyFactory factory = SecretKeyFactory.getInstance("PBKDF2WithHmacSHA1");
        byte[] inputHash = factory.generateSecret(spec).getEncoded();
        return Arrays.toString(inputHash);
    }

    public Package hash(String password) throws NoSuchAlgorithmException, InvalidKeySpecException {
        SecureRandom random = new SecureRandom();
        byte[] salt = new byte[16];
        random.nextBytes(salt);

        KeySpec spec = new PBEKeySpec(password.toCharArray(),salt,65536, 128);
        SecretKeyFactory factory = SecretKeyFactory.getInstance("PBKDF2WithHmacSHA1");
        byte[] hash = factory.generateSecret(spec).getEncoded();
        return new Package(Arrays.toString(hash),65536,salt);
    }


About the project

Politykle is an educational game designed to engage users in learning about Polish politics through an interactive and entertaining experience. Inspired by the popular game format of loldle.net, Politykle offers three distinct game modes, each focusing on different aspects of Polish political knowledge, including politicians, political parties, and election results. The project was developed with the goal of educating users in preparation for the upcoming 2025 Presidential Elections.

    Game Modes:
  • Guess the Politician: The game randomly selects a politician based on the chosen difficulty level (Easy, Medium, or Hard). Players guess the politician by comparing attributes such as age, political party, and other criteria. Feedback is provided for each guess, indicating whether the player is correct, needs to guess higher, or lower.
  • Guess the Party:
    The game randomly selects a political party and presents the player with a politician from that party. Players must guess the party by entering its abbreviation.
  • Guess the Election:
    The game randomly selects an election event and displays a political party along with its election result. Players guess the election based on the provided information.
  • A simple, intuitive user interface

Technical Implementation

The project features a robust and normalized relational database designed to store and manage data efficiently. The database is normalized to the Third Normal Form (3NF), ensuring minimal redundancy and maximum consistency. Key components include:

  • Entities
  • Relationships

Triggers and security

  • Before Delete Triggers: Automatically remove related records from junction tables when a politician, party, position, or event is deleted.
  • Password Hashing: User passwords are securely stored using salted hashes.
  • SQL Injection Prevention: Parameterized queries are used to prevent SQL injection attacks.
  • Access Control: Two user roles are supported: Administrators: Full CRUD (Create, Read, Update, Delete) access to all tables. Users: Read-only access to data necessary for gameplay.
