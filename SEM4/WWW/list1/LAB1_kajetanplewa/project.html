public BoardBase build() {
    BoardBase boardBase = new BoardBase();
    PoleStorage poleList = new PoleStorage();
    findMaxPoles(this.layers);
    Pole current= creator.createInitPole();
    current.setColor(Colors.Grey);
    poleList.insert(current);
    for(int i=0;i< maxPoles;i++){
        current=poleList.getByIndex(i);
        addNeighbours(current, poleList);
    }
    boardBase.setStorage(poleList);
    return boardBase;
}

public void addNeighbours(Pole currentPole, PoleStorage poleStorage){
    for (Directions direction : possibleMoves) {
    Pole temp = creator.createBasePole(currentPole, direction, this.layers);
    if(temp==null){
        continue;}
    if (poleStorage.contains(temp)) {
        Pole existingPole = poleStorage.get(temp);
        currentPole.addNeighbour(existingPole);
    } else {
        poleStorage.insert(temp);
        currentPole.addNeighbour(temp);
        }
    }
}


About the project

The Chinese Checkers project is an implementation of the popular board game in Java, which I created together with Janek Konieczny. The application allows for multiplayer gameplay thanks to its client-server architecture. Key features include:

  • Support for multiple concurrent games
  • Client-server communication
  • Multiplayer gameplay
  • A simple, intuitive user interface

Technologies used:

  • Java (object-oriented programming)
  • Sockets (network communication)
  • GUI in Java Swing

The presented algorithm builds the board by creating objects representing the tiles (Tile) and connecting them as neighbors. This process takes place in two main stages:

  • Initialization and filling the tile collection: The algorithm starts by creating the first, initial tile, and then iteratively generates the remaining tiles, adding neighbors to them.
  • Connecting the tiles: For each tile in the collection, all possible movement directions are checked (possibleMoves). For each direction, it tries to create a neighboring tile. If the tile in that direction already exists, it is simply connected as a neighbor; otherwise, a new tile is created, added to the collection, and connected as a neighbor.
